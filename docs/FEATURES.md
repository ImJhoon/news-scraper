# 뉴스 스크래퍼(News Scraper) 프로젝트 가이드 및 구조

이 문서는 뉴스 스크래퍼 프로젝트의 전반적인 구조와 각 클래스/파일의 역할을 처음 프로젝트를 보는 사람도 이해하기 쉽도록 아주 자세하게 정리한 문서입니다. 

객체지향 프로그래밍(OOP) 원칙과 클린 아키텍처 관점에서 코드를 어떻게 나누었는지, 계층(도메인, 애플리케이션, 인프라스트럭처)별로 어떤 역할을 하는지 설명합니다.

---

## 1. 도메인 계층 (Domain Layer)
도메인 계층은 프로젝트의 핵심 비즈니스 개념(데이터 모델, 규칙)을 정의하는 곳입니다. 외부 기술이나 프레임워크에 의존하지 않고 프로젝트의 중심 뼈대 역할을 합니다.

### 1.1 `NewsResult` (뉴스 데이터 모델)
* **어떤 파일인가요?**: 검색된 뉴스 기사 한 건의 데이터를 담아 전달하는 그릇(Data Transfer Object, DTO) 역할을 합니다.
* **왜 이렇게 만들었나요?**: 
  * 자바 16부터 도입된 `record` 키워드를 사용했습니다. `record`를 사용하면 생성자나 값을 꺼내오는 메서드를 일일이 만들지 않아도 코드가 매우 간결해집니다.
  * 한 번 생성된 데이터는 중간에 마음대로 바뀌면 버그를 찾기 어려우므로, 데이터 수정을 막는 '불변 객체(Immutable)'로 설계하기 위해 사용했습니다.
* **어떤 데이터를 가지나요?**: 기사 제목(`title`), 본문 요약(`description`), 기사 원문 링크(`url`), 발행일자(`pubDate`).

### 1.2 `NewsCategory` (검색 정렬 기준)
* **어떤 파일인가요?**: 뉴스를 검색할 때 어떤 기준으로 정렬할지(정확도순, 최신순)를 미리 정의해 둔 열거형(Enum) 클래스입니다.
* **왜 이렇게 만들었나요?**: 문자열(예: `"sim"`, `"date"`)을 그대로 코드에 쓰다 보면 오타가 날 수도 있고, 어떤 값이 들어갈 수 있는지 파악하기 어렵습니다. `Enum`으로 만들어두면 정해진 선택지(`SIM`, `DATE`) 안에서만 안전하게 사용할 수 있습니다.
* **기능**: 네이버 API가 요구하는 쿼리 파라미터 값(`sim`, `date`)과 사람이 읽기 편한 한글 설명("정확도순", "최신순")을 매칭하여 제공합니다.

## 2. 애플리케이션 계층 (Application Layer)
애플리케이션 계층은 비즈니스 흐름(워크플로우)을 통제하는 곳입니다. 도메인 데이터를 활용하여 '뉴스 검색 -> 처리 -> 발행(출력)'이라는 작업 순서를 지휘합니다.

### 2.1 `NewsProvider` (뉴스 제공자 인터페이스)
* **어떤 파일인가요?**: "뉴스를 어디선가 검색해서 가져온다"는 행동을 정의해 둔 설계도(인터페이스)입니다.
* **왜 이렇게 만들었나요?**: 지금은 네이버에서 뉴스를 가져오지만, 나중에는 구글 검색이나 다음 웹문서에서 가져올 수도 있습니다. 이런 변화에 코드를 모두 뜯어고치지 않고 유연하게 대처하기 위해 추상화된 껍데기만 만들어 두었습니다.
* **기능**: 검색어(`searchQuery`)와 가져올 개수(`limit`)를 넘기면 `NewsResult` 형태의 목록을 반환하는 `fetchNews` 메서드를 가집니다.

### 2.2 `NewsPublisher` (뉴스 발행자 인터페이스)
* **어떤 파일인가요?**: "수집된 뉴스를 어딘가로 내보낸다"는 행동을 정의해 둔 설계도입니다.
* **왜 이렇게 만들었나요?**: 뉴스 데이터를 콘솔 화면에 단순 출력할지, 엑셀 파일로 저장할지, 슬랙(Slack) 메신저로 쏠지 아직 결정되지 않았거나 수시로 바뀔 수 있으므로 인터페이스로 분리해 두었습니다.
* **기능**: 검색 주제(`topic`)와 수집된 뉴스 목록(`newsResults`)을 받아서 최종 처리하는 `publish` 메서드를 가집니다.

### 2.3 `NewsService` (뉴스 서비스 로직)
* **어떤 파일인가요?**: 실제로 일을 지휘하는 '오케스트라 지휘자' 역할의 클래스입니다.
* **어떻게 일하나요?**:
  1. `NewsProvider`에게 지시하여 뉴스를 가져오라고 합니다.
  2. 가져온 결과를 `NewsPublisher`에게 넘겨 발행(출력/저장 등)을 지시합니다.
* **특징 (의존성 주입)**: 이 클래스는 자기가 직접 네이버 API를 찌르거나 화면에 글씨를 쓰지 않습니다. 외부에서 조립된 Provider와 Publisher 부품을 전달받아(생성자로 주입받아) 사용하므로 코드가 아주 깔끔하고 유지보수가 쉽습니다.

## 3. 인프라스트럭처 계층 (Infrastructure Layer)
가장 바깥쪽에 위치하며, 외부 API, 데이터베이스, 네트워크 등과의 통신을 직접 담당하는 구체적인 기술 계층입니다.

### 3.1 `AbstractHttpScraper` (HTTP 통신 공통 클래스)
* **어떤 파일인가요?**: 웹(HTTP)을 통해 외부 API와 통신할 때 필요한 공통적인 도구들을 모아둔 추상 클래스입니다.
* **왜 이렇게 만들었나요?**: 네이버 뉴스뿐만 아니라 나중에 다른 웹사이트나 API를 연동할 때도 통신 기능이 중복으로 필요하므로, 재사용할 수 있도록 기반(Base) 클래스로 만들었습니다.
* **주요 도구**: Java 11 이상에서 기본 제공하는 최신 통신 모듈인 `HttpClient` 객체를 내부에 가지고 있어, 상속받는 자식 클래스들이 편리하게 네트워크 요청을 할 수 있습니다.

### 3.2 `NaverNewsProvider` (네이버 뉴스 연동 구현체)
* **어떤 파일인가요?**: `NewsProvider` 인터페이스(규칙)를 따르고 `AbstractHttpScraper`를 상속받아 네이버 오픈 API와 직접 소통하는 구체적인(실제 동작하는) 클래스입니다.
* **어떻게 일하나요?**:
  1. **환경 변수 읽기**: API 통신에 필요한 민감한 정보(Client ID, Secret Key)를 소스 코드에 대놓고 적지 않고, 안전하게 컴퓨터의 환경 변수(`System.getenv`)에서 읽어옵니다.
  2. **URL 생성 및 인코딩**: 한글 검색어가 깨지지 않도록 `URLEncoder`를 사용해 안전하게 변환하고, 네이버 API 주소 뒤에 검색 파라미터(`query`, `display`, `sort` 등)를 붙여 완성된 URL을 조립합니다.
  3. **요청 및 응답 처리**: 네이버가 요구하는 인증 헤더(`X-Naver-Client-Id` 등)를 HTTP GET 요청에 담아 전송한 뒤, 응답으로 온 문자열(JSON 형식의 뉴스 데이터)을 받아옵니다.
  4. **데이터 파싱 (수동)**: 응답받은 순수 텍스트(JSON) 문자열을 `String.split()`을 활용한 `cutText` 헬퍼 메서드로 직접 잘라내어(파싱), 필요한 항목(`title`, `link`, `description` 등)만 추출한 뒤 `NewsResult` 객체로 변환하여 리스트에 담아 반환합니다. (외부 라이브러리 없이 순수 Java 문법으로 처리)

### 3.3 `ConsoleNewsPublisher` (콘솔 출력 구현체)
* **어떤 파일인가요?**: `NewsPublisher` 인터페이스를 구현하여, 수집된 뉴스 데이터를 콘솔 화면(터미널)에 예쁘게 출력해주는 클래스입니다.
* **어떻게 일하나요?**:
  * 전달받은 검색 주제(`topic`)와 `NewsResult` 리스트를 순회하며 화면에 출력합니다.
  * Java 15부터 도입된 **텍스트 블록(Text Blocks, `"""..."""`)**과 `String.formatted()`를 사용하여 가독성 좋고 깔끔하게 여러 줄의 문자열을 포맷팅하여 출력합니다.

## 4. 환경 및 설정 (Configuration)

### 4.1 `.env` & `.env.sample` (API 키 및 설정 관리)
* **어떤 파일인가요?**: 프로젝트 실행에 필수적인 비밀번호, API 인증 키 등을 파일 하나로 관리하기 위한 환경 변수 파일입니다.
* **왜 이렇게 관리하나요? (보안)**:
  * API 키가 코드로 작성되어 Github 같은 공개 저장소에 올라가면 해킹이나 과금 폭탄의 위험이 있습니다. 따라서 진짜 키가 들어있는 `.env` 파일은 `.gitignore`에 등록하여 **절대 업로드되지 않게** 막아두었습니다.
  * 대신 다른 개발자나 사용자가 "이 프로젝트를 돌리려면 어떤 설정값들이 필요한지" 파악할 수 있도록 뼈대만 있는 `.env.sample` 파일을 제공합니다. (실제 값 대신 `******` 마스킹 처리됨)
